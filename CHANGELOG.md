# Changelog

모든 버전별 주요 변경 사항 및 업데이트 내역을 이 파일에 기록합니다.

## [1.3.0] - 2026-02-22

### Added
- **결정론적 시각화 렌더링 (Deterministic Rendering)**
  - `toSeedNumber` 로직을 도입하여 화면 새로고침 시에도 도형 위치, 애니메이션 딜레이 등이 일관되게 유지됨
- **측정 및 비교 모듈 고도화**
  - `MeasurementLength` 컴포넌트 신설 및 `buildMeasurementVisual` 로직 강화
  - 비교하기(`compare`) 단원에 막대 차트 연동 및 시각적 피드백 개선
- **품질 관리 (Quality Assurance)**
  - 테스트 케이스 4개 추가 (총 43개 Pass)
  - `TopicHasVisualSupport` 구조를 통한 시각화 우선순위 최적화

## [1.2.1] - 2026-02-21

### Fixed
- **문제 생성 로직 불균형 수정**
  - 비산술 단원(비교하기, 길이 재기 등) 진입 시 시각화 문제가 우선 출제되도록 개선
  - `MeasurementLength` 시각화 컴포넌트 추가 및 렌더링 최적화
- **결정론적 렌더링 도입**
  - 시각화 요소의 무작위 배치를 Seed 기반으로 일관성 있게 개선
- **테크니컬 스택 고도화**
  - 테스트 프레임워크를 `Vitest`로 전환하여 개발 생산성 및 안정성 확보

## [1.1.0] - 2026-02-21

### ✨ 학습 모드 고도화 및 보상 시스템
- **지능형 연산 가이드 도입**: 덧셈/뺄셈 시 받아올림(+1) 및 받아내림(↘1)을 자동으로 표시하는 교육용 UI 탑재.
- **단원 완료 축하 효과**: 모든 문제 풀이 완료 시 화면 가득 쏟아지는 **Confetti(폭죽) 애니메이션** 추가.
- **학습 동기부여 강화**: 
  - **스트릭(Streak)**: 연속 학습 일수(🔥) 표시 시스템 도입.
  - **완료 배지**: 단원 선택 화면 내 완료된 항목에 대한 **🏅 메달 아이콘** 표시.
- **인터랙티브 튜토리얼**: 단원을 단계별 슬라이드로 나누어 개념을 더 상세히 설명하도록 개선.

## [1.0.1] - 2026-02-21

### 🚀 학습 모드 대폭 개편
- **목차 시스템(Curriculum) 도입**: 단순히 무작위 문제를 푸는 방식에서 벗어나, 주제별(덧셈 기초, 구구단 등) 학습 로직 구축.
- **개념 설명(Tutorial) 화면 추가**: 단원 진입 전 핵심 원리를 이모지와 시각적 텍스트로 설명하는 튜토리얼 기능 도입.
- **자릿수별 단계적 풀이(Semi-step) 모드**: 큰 수의 연산 시 '일의 자리', '십의 자리' 등 자릿수별로 집중해서 입력할 수 있는 스마트 입력 UI 탑재.
- **학습 이어하기 기능**: 사용자의 학습 진행 상황을 자동으로 저장하여, 중단된 시점부터 다시 공부할 수 있는 시스템 구현.
- **상시 힌트 시스템**: 오답 시 노출되던 힌트를 상시 버튼으로 배치하여 개념 이해도 향상.

## [1.0.0] - 2026-02-21

### 🎉 신규 릴리즈 (Initial Release)
- **프로젝트 뼈대 및 디자인 시스템 구축**
  - Vite + React 19 환경 초기화.
  - 최신 디자인 트렌드인 글래스모피즘(Glassmorphism) UI 및 마이크로 애니메이션(Bouncing, Shake, Pop) 적용.
  - 모바일 친화적인 자체 숫자 키패드(`Keypad`) 컴포넌트 개발.
- **수학 문제 생성 엔진 릴리즈**
  - 레벨 1 ~ 4 난이도에 따른 덧셈, 뺄셈, 곱셈, 나눗셈(나머지 없음) 난수 문제 생성 로직 개발.
- **학습 모드 출시**
  - 자가 주도적 학습이 가능한 무제한 시간 풀이 모드.
  - 연속 오답 시 🍎 사과 이모지 및 식 풀이를 제공하는 '단계별 시각적 힌트' 도입.
- **퀴즈 모드 출시**
  - 제한 시간(Timer) 압박 속에서 긴장감 있게 즐기는 모드.
  - 연속 정답 시 콤보(Combo) 알람 및 팝업 이펙트 추가.
- **복습 모드 (나만의 오답 노트) 출시**
  - 브라우저 데이터(LocalStorage)를 활용한 틀린 문제 자동 수집.
  - 기존 문제를 바탕으로 숫자 범위, 패턴의 일의 자리를 10~20% 변형해서 재출제하는 '유사 변형 문제 알고리즘' 도입.
